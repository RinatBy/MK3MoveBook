# MK3 MoveBook

Автономная русскоязычная интерактивная книга приёмов Ultimate Mortal
Kombat 3. Desktop-приложение открывает локальный интерфейс через Microsoft
Edge WebView2 и не требует запущенной игры.

## Возможности

- 26 бойцов и две версии набора приёмов;
- разделы «Приёмы», «История», «Факты» и «Секреты и коды»;
- обозначения кнопок UMK3 и SEGA;
- поиск по бойцам и приёмам;
- локальные видео демонстраций;
- обновление содержимого книги через GitHub Releases;
- проверка SHA-256 и автоматический возврат к встроенной версии при ошибке.

## Запуск готовой версии

1. Скачайте архив из раздела Releases.
2. Распакуйте его в отдельную папку.
3. Запустите `MK3 MoveBook.exe`.

Если WebView2 недоступен, запустите `Web MK3 MBook.exe`. Локальная
веб-версия откроется в браузере, выбранном в Windows по умолчанию.

EXE, DLL-файлы и папка `web` должны оставаться рядом. Для работы требуется
Microsoft Edge WebView2 Runtime, который обычно уже установлен в Windows 10
и Windows 11. Если компонент отсутствует, приложение предложит запустить
официальный Bootstrapper Microsoft из полного архива.

Веб-версия не заменяет локальные файлы автоматически. Кнопка обновления
открывает страницу последнего релиза для ручного скачивания.

## Разработка

Главный источник web-интерфейса находится в `web`. Desktop-хост:
`desktop/MoveBookHost.cs`; браузерный запуск:
`desktop/MoveBookWebLauncher.cs`.

### Платформенная модель данных

`web/data/moves.js` остаётся единой аркадной базой приёмов. Объект
`platformModel` добавляет поверх неё доступность для `arcade` и `sega`:

- `fighters.default` задаёт обычную доступность бойца, а
  `fighters.overrides` хранит только исключения и примечания;
- `moves.default` считает существующий приём аркадным и не подтверждает его
  для SEGA до ручной проверки;
- `moves.rules` описывает общие проверенные исключения, например отсутствие
  Animality в стандартной версии для SEGA;
- конкретный боец или приём может переопределить модель полями `platforms`,
  `platformStatus` и `platformNotes`.

Для подтверждённого отличия приёма на SEGA в сам приём добавляется
`platformOverrides.sega`. В нём можно независимо указать `notation`,
`requirements` и `video`, не копируя остальные данные бойца:

```js
{
  "label": "Example",
  "notation": "_4,_6,HP",
  "platforms": ["arcade", "sega"],
  "platformOverrides": {
    "sega": {
      "notation": "_4,_6,LP",
      "requirements": "Условие для версии SEGA",
      "video": "assets/animations/sega/example.webm"
    }
  }
}
```

### Прямые адреса и SEO-метаданные

Интерфейс распознаёт адреса `/arcade/<fighter>/` и `/sega/<fighter>/`.
В Portable сохраняется совместимая hash-навигация, поэтому локальные файлы
продолжают открываться без веб-сервера. Для каждого бойца используются:

- существующее имя в книге как единственный видимый `h1`;
- русское SEO-имя из `moves.js`;
- динамические `title` и `meta description` с учётом платформы;
- отдельное честное описание, если боец недоступен в выбранной версии.

Статические страницы по этим адресам создаются генератором и затем могут
публиковаться в GitHub Pages.

Генерация полного сайта:

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\generate-site.ps1 `
  -WebRoot .\web `
  -OutputRoot .\.site-dist `
  -BaseUrl https://rinatby.github.io/MK3MoveBook/
```

Скрипт создаёт главную, страницы платформ и бойцов, `robots.txt`,
`sitemap.xml`, canonical и Open Graph. В HTML заранее находятся видимые H1,
список бойцов и приёмы; JavaScript поверх них включает интерактивную книгу.
Каталог `.site-dist` является проверочным результатом и не коммитится.

Сборка EXE:

```powershell
powershell -ExecutionPolicy Bypass -File .\desktop\build.ps1
```

Подготовка релизных архивов и манифеста:

```powershell
powershell -ExecutionPolicy Bypass -File .\desktop\create-release.ps1 -Version 1.0.0
```

Обновлятор загружает только новую папку `web`. Предыдущая встроенная версия
не перезаписывается и остаётся аварийным вариантом запуска.

Репозиторий хранит исходный код интерфейса и desktop-хоста. Готовый пакет с
необходимыми локальными ресурсами публикуется отдельно в разделе Releases.
